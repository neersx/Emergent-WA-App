2 Install Prerequisites

2.1 — Homebrew

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Apple Silicon — add to PATH:

echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile && source ~/.zprofile

2.2 — Python 3.11

brew install python@3.11
echo 'export PATH="/opt/homebrew/opt/python@3.11/bin:$PATH"' >> ~/.zprofile && source ~/.zprofile

2.3 — Node.js 20 LTS

brew install node@20
echo 'export PATH="/opt/homebrew/opt/node@20/bin:$PATH"' >> ~/.zprofile && source ~/.zprofile

2.4 — Yarn

npm install -g yarn

2.5 — MongoDB 7.0

brew tap mongodb/brew
brew install mongodb-community@7.0
brew services start mongodb-community@7.0

Verify: mongosh --eval "db.runCommand({connectionStatus:1})" --quiet → should print { ok: 1 }

Alternative — Docker: docker run -d --name mongodb -p 27017:27017 mongo:7.0

3 Clone the Repository

git clone https://github.com/neersx/Emergent-WA-App.git
cd Emergent-WA-App

4 MongoDB Setup

# Verify MongoDB is running
brew services list | grep mongodb
# Expected: mongodb-community  started

The app creates the whatsapp_saas database automatically on first run.

5 Backend Setup

cd backend

# Create virtual environment
python3.11 -m venv venv

# Activate it
source venv/bin/activate   # (venv) should appear in your prompt

# Install dependencies
pip install --upgrade pip
pip install -r requirements.txt
Install time: ~2–3 minutes

6 Frontend Setup

cd frontend
yarn install
Install time: ~1–2 minutes. Peer dependency warnings are harmless.

7 Environment Variables

Run the automated setup script (generates secure cryptographic keys):

chmod +x scripts/setup-env.sh
./scripts/setup-env.sh

Or create them manually:

backend/.env

MONGO_URL=mongodb://localhost:27017
DB_NAME=whatsapp_saas
CORS_ORIGINS=http://localhost:3000
JWT_SIGNING_KEY=<run: python3 -c "import secrets; print(secrets.token_hex(32))">
TOKEN_ENCRYPTION_KEY=<run: python3 -c "import base64,secrets; print(base64.urlsafe_b64encode(secrets.token_bytes(32)).decode())">
META_APP_ID=
META_APP_SECRET=
META_EMBEDDED_SIGNUP_CONFIG_ID=
META_WEBHOOK_VERIFY_TOKEN=local-dev-token
META_GRAPH_API_VERSION=v21.0
META_MOCK_MODE=true
MEDIA_STORAGE_MODE=disk
MEDIA_STORAGE_PATH=./media
WORKER_ENABLED=true
WORKER_POLL_INTERVAL_MS=500

frontend/.env

REACT_APP_BACKEND_URL=http://localhost:8001
Never commit .env to Git — it's already in .gitignore

8 Run the Application

Option A — One Command (Recommended)

chmod +x scripts/start.sh
./scripts/start.sh

Starts MongoDB + Backend + Frontend, tails all logs in one terminal. Press Ctrl+C to stop everything.

Option B — Make

make install   # first time only
make dev       # start all services (background)
make logs      # view output
make stop      # stop all

Option C — 3 Separate Terminals

Terminal 1 — Backend

cd backend
source venv/bin/activate
uvicorn server:app \
  --host 0.0.0.0 \
  --port 8001 \
  --reload

Terminal 2 — Frontend

cd frontend
yarn start

10 Meta API — Live Mode

Mock mode (META_MOCK_MODE=true) works fully without Meta credentials. Skip this section until you're ready to go live.

Step 1 — Get credentials from Meta

Go to developers.facebook.com/apps and find:

VariableLocation in Meta Dashboard
META_APP_IDSettings → Basic → App ID
META_APP_SECRETSettings → Basic → App Secret
META_EMBEDDED_SIGNUP_CONFIG_IDWhatsApp → Embedded Signup → Config ID
META_WEBHOOK_VERIFY_TOKENYour own chosen secret string

Step 2 — Expose localhost with ngrok

brew install ngrok/ngrok/ngrok
ngrok http 8001
# Copy the https://xxxxx.ngrok.io URL

Step 3 — Set webhook in Meta Dashboard

Callback URL: https://xxxxx.ngrok.io/api/webhooks/meta
Verify Token: (same as your META_WEBHOOK_VERIFY_TOKEN)

Step 4 — Update backend/.env

META_APP_ID=your_app_id
META_APP_SECRET=your_app_secret
META_EMBEDDED_SIGNUP_CONFIG_ID=your_config_id
META_WEBHOOK_VERIFY_TOKEN=your_verify_token
META_MOCK_MODE=false

Restart the backend after changing .env.

11 WebSocket Verification

# Install wscat
npm install -g wscat

# After logging in via browser, test WebSocket:
wscat -c "ws://localhost:8001/api/ws/inbox"

The server sends {"type":"auth_required"} — the browser sends the cookie automatically. In the browser DevTools → Network → WS tab, you should see the connection and live events when you simulate an inbound message.

14 Troubleshooting

❌ MongoDB: ServerSelectionTimeoutError localhost:27017
brew services start mongodb-community@7.0
❌ error: legacy-install-failure (bcrypt / cryptography)
xcode-select --install → then retry pip install -r requirements.txt
❌ Port 3000 already in use
kill -9 $(lsof -ti:3000)
❌ Port 8001 already in use
kill -9 $(lsof -ti:8001)
❌ TOKEN_ENCRYPTION_KEY not a valid Fernet key
python3 -c "import base64,secrets; print(base64.urlsafe_b64encode(secrets.token_bytes(32)).decode())"
❌ CORS error in browser
Set CORS_ORIGINS=http://localhost:3000 in backend/.env and restart backend.
❌ Apple Silicon: yarn install fails (native modules)
rm -rf node_modules && arch -arm64 yarn install
❌ Template sync fails with 401
Expected — the seeded demo WABAs have a fake token. Only real WABAs from Embedded Signup will sync.
